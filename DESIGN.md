# LOF Arbiter - 设计文档

> 详细记录 LOF Arbiter 的设计决策、交互规范与技术实现。

---

## 1. 项目背景

### 1.1 目标
为普通投资者提供 LOF 基金折溢价套利的智能咨询，通过自然语言对话形式，降低套利策略的理解门槛。

### 1.2 核心价值
- **信息聚合**：将分散的行情、净值、费率数据融合呈现
- **智能筛选**：自动识别高溢价/折价机会
- **风险提示**：帮助用户理解套利成本与风险

---

## 2. 数据架构

### 2.1 DataHub 依赖

LOF Arbiter 依赖 DataHub ETL Pipeline 产出的融合表：

```
ods_fund_etf_category_sina  ─┐
ods_fund_open_daily         ─┼─→  dwd_fund_lof
ods_fund_purchase           ─┘
```

### 2.2 数据更新频率

| 数据源 | 更新频率 | 说明 |
|--------|----------|------|
| LOF 实时行情 | 交易时段实时 | 新浪财经 |
| 基金净值 | 收盘后更新 | 东方财富 T+1 |
| 溢价率 | 实时计算 | = (现价 - 净值) / 净值 |

### 2.3 关键计算

**溢价率计算**：
```
溢价率(%) = (现价 - 净值) / 净值 × 100
```

**套利收益估算**（买入套利）：
```
收益 = 净值涨幅 - 申购费 - 赎回费 - 交易佣金
```

**费用参考**：
| 费用项 | 典型值 |
|--------|--------|
| 申购费 | 1.2% ~ 1.5% |
| 赎回费 | 0.5%（持有≥7天），1.5%（<7天） |
| 交易佣金 | 万三 ~ 万一 |

## 3. 功能定义

### 3.1 套利机会查询

**触发方式**：
- "今天有什么套利机会？"
- "哪些 LOF 溢价率高？"
- "帮我看看折价的 LOF"

**返回内容**：
1. 高溢价 TOP10（适合卖出赎回）
2. 高折价 TOP10（适合买入套利）
3. 每个品种标注：溢价率、现价、净值、成交额
4. 风险提示

**筛选规则**：
- 成交额 > 100万（流动性门槛）
- 申购状态 = "开放式"

### 3.2 单基金诊断

**触发方式**：
- "帮我看看 160140"
- "XX基金怎么样？"
- "161628 能买吗？"

**返回内容**：
1. 基本信息：名称、代码、现价、净值
2. 溢价率（含颜色标识）
3. 流动性：成交额、近几日趋势
4. 操作建议：买入/卖出/观望

### 3.3 操作建议逻辑

| 条件 | 建议 |
|------|------|
| 溢价率 > 1.5% | 可考虑卖出赎回 |
| 折价率 < -1.5% 且成交额大 | 可考虑买入 |
| 折溢价率在 ±0.5% 以内 | 无明显套利空间，观望 |
| 暂停申购 | 无法套利，提示风险 |

### 3.4 收益测算

**触发方式**：
- "套利收益怎么算？"
- "买入 10 万能赚多少？"
- "帮我算算 160140 的套利收益"

**输入参数**：
- 买入金额
- 持有天数

**输出内容**：
- 买入费用明细
- 持有期间净值变化预估
- 卖出费用明细
- 理论净收益

## 4. 交互规范

### 4.1 回复格式

**结构化回复**：
- 使用 emoji 增强可读性
- 数据用表格/列表清晰呈现
- 操作建议用颜色标注风险等级

**回复示例**：
```
📊 今日 LOF 套利机会

🔥 高溢价 TOP5（卖出赎回套利）
| 基金名称 | 代码 | 溢价率 | 现价 | 净值 | 成交额 |
|---------|------|--------|------|------|--------|
| 招商全球  | 160140 | +2.35% | 1.250 | 1.222 | 3200万 |
| 国泰互联  | 161628 | +1.87% | 0.980 | 0.962 | 1800万 |
...

⚠️ 风险提示：
- 套利需 T+2 交割，资金占用两天
- 赎回费通常 0.5%，持有 <7天 为 1.5%
- 高溢价可能因流动性不足无法成交
```

### 4.2 异常处理

| 情况 | 回复策略 |
|------|----------|
| 数据为空 | "今日暂无 LOF 行情数据，可能已收盘" |
| 数据库连接失败 | "数据服务暂时不可用，请稍后再试" |
| 基金代码不存在 | "未找到该基金，请检查代码是否正确" |
| 非交易时间 | "当前非交易时段，数据可能为昨日收盘价" |

### 4.3 上下文记忆

- 记住用户最近查看的基金
- 连续提问时可省略基金名称
- 例："帮我看看 160140" → "现在多少？" → 理解为同一只基金

## 5. 技术实现

### 5.1 Skill 结构

```

├── SKILL.md          # Skill 定义文件
├── scripts/
│   ├── query.py      # 数据查询脚本
│   └── calculate.py  # 溢价率计算

│   └── config.json   # 配置文件
└── README.md
```

### 5.2 数据库访问

通过 DataHub 的 SQLite 数据库直接查询：

```python
import sqlite3

def get_lof_data():
    conn = sqlite3.connect('/path/to/datahub.db')
    df = pd.read_sql_query(
        "SELECT * FROM dwd_fund_lof WHERE 交易日期 = ?",
        conn,
        params=[today]
    )
    conn.close()
    return df
```

### 5.3 依赖关系

- Python 3.9+
- pandas
- sqlite3（标准库）
- akshare（数据源，DataHub 已依赖）
